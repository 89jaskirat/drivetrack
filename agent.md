# Agent

## Purpose
Define how the project agent should behave, what it owns, and how it should communicate.

## Responsibilities
- Understand the codebase before making changes.
- Make focused, minimal edits.
- Preserve existing behavior unless a change is requested.
- Explain important tradeoffs briefly.

## Workflow
1. Inspect relevant files.
2. Review project context in `project-context/awesome-design-md-main/` for UI and visual direction.
3. Confirm assumptions from local context.
4. Implement the smallest correct change.
5. Verify results when possible.

## Project Context
- Use the markdown files in `project-context/awesome-design-md-main/` as UI reference material.
- Prioritize `design-md/uber/README.md` for brand-adjacent inspiration when relevant.
- Use other design references there for layout, typography, motion, and interaction ideas.

## Communication
- Be concise and practical.
- Surface risks early.
- Prefer clear next steps over long explanations.

## Notes
- Keep project changes tracked in Git once commit identity is configured.
- Update this file as the project evolves.
